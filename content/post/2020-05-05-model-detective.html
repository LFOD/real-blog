It has recently been brought to our attention that a model is predicting we will have 0 COVID-19 related deaths after May 15th. This model was described as a ‚Äúcubic polynomial‚Äù. Inspired by Dave Robinson‚Äôs data sleuthing, I decided to see if we could reverse engineer this ‚Äúmodel‚Äù.

The ‚Äúcubic model‚Äù from @CEA + Kevin Hassett was pretty clearly fit on log(deaths + 1). Which is‚Ä¶ pretty dangerous for forecasting.

Just imagine if they'd fit a quartic model pic.twitter.com/VTlFoD9qvC

Turns out we can!

Let‚Äôs read in the data from the New York Times GitHub Repo.

library(tidyverse)
library(magick)
library(grid)

img <- image_read("https://pbs.twimg.com/media/EXRujkPWsAEHdx4?format=jpg&name=medium")
d <- read_csv("https://raw.githubusercontent.com/nytimes/covid-19-data/master/us.csv")

I‚Äôm going to do some data manipulation:

  1. Add some future dates for our ‚Äúprediction model‚Äù
  2. Filter so we only include dates from the graph we are trying to replicate
  3. The New York Times gives us cumulative deaths, we want just deaths per day, so I‚Äôll create that
dat <- d %>%
  bind_rows(tibble(date = Sys.Date() + 1:91)) %>%
  filter(date >= "2020-02-26") %>%   
  mutate(death = c(deaths[1], diff(deaths)))

Here is a VERY basic prediction model. It includes a 3rd degree polynomial term it is predicting log(death + a small constant). I tried a few, a small constant of 0.5 seems to fit the best ü§∑.

p <- predict(lm(log(death + 0.5) ~ poly(date, 3), data = dat), newdata = dat)

Let‚Äôs plot it. First I‚Äôll create a plot that just includes the original graph.

p1 <- ggplot(dat, aes(x = date, y = death)) +
  geom_line() + 
  annotation_custom(rasterGrob(img, 
                               width = unit(1,"npc"), 
                               height = unit(1,"npc")), 
                    -Inf, Inf, -Inf, Inf) +
  ylim(c(-500, 5000)) +
  theme(axis.title = element_blank(), 
        axis.text = element_blank(),
        axis.ticks = element_blank()) + 
  scale_x_date(limits = as.Date(c('2020-02-17','2020-08-10')))
p1

Now I‚Äôll add a plot that has the New York Times death data overlaid, just to be sure things line up properly.

p2 <- ggplot(dat, aes(x = date, y = death)) +
  geom_line() +

p2

Finally, let‚Äôs plot our ‚Äúmodel‚Äù.

p3 <- ggplot(dat, aes(x = date, y = death)) +
  annotation_custom(rasterGrob(img, 
                               width = unit(1,"npc"), 
                               height = unit(1,"npc")), 
                    -Inf, Inf, -Inf, Inf) +
  geom_line() + 
  geom_line(aes(x = date, y = exp(p))) +
  ylim(c(-500, 5000)) +
  theme(axis.title = element_blank(), 
        axis.text = element_blank(),
        axis.ticks = element_blank()) + 
  scale_x_date(limits = as.Date(c('2020-02-17','2020-08-10')))
p3
