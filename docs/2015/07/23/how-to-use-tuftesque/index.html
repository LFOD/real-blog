The Begining

If you already have a blogdown site than you can skip to the next step. If you do not, open R in a new fresh directory and run the following commands:

#downloads the blogdown package
devtools::install_github("rstudio/blogdown")

#initialize a new blog
blogdown::new_site()

Boom, now you have your very own blog! Wouldn’t it be nice if everything in life was this easy?

Adding tuftesque

Now that you have your blogdown blog up and running switch back into your R terminal (you may have to press escape to kill the running blogdown server) and run the following command to download the tuftesque theme from github.

blogdown::install_theme('nstrayer/tuftesque')

Assuming this successfully worked (if it didn’t file an issue here) now simply run the command blogdown::serve_site() to restart the blogdown server and you should have your fancy new blog working!

Banner and Logo

In the config.toml for your site set both params.logo and params.banner with a key of url and point it to whatever you desire your logo to be. This can also be a local file in the static directory, as is the case below for the banner.

E.g.

[params]
  description = "A website built through Hugo and blogdown."
 
  [params.logo]
    url = "https://pbs.twimg.com/profile_images/781518772259749888/lmEHgoQP_400x400.jpg"
  
  [params.banner]
    url = "/images/banner.png"

Banner Size: The banner image defaults to being a 3:1 ratio of width to height, so if your image is that size it will be fully shown. Otherwise some will get cut off.

If you want to have a custom html banner like the one on livefreeordichotomize this can be done by creating the file layouts/partials/banner.html and filling it with whatever you would desire. Here’s an example of the one from the blog above. By placing this file here hugo/blogdown will automatically use it instead of the default photo.

Side Notes:

We all know why you’re here: the sidenotes. This is rather easy. Just make sure you have the tufte package installed and then, in an inline r chunk, wherever you want to put a side note call the following.

tufte::margin_note("This is a margin note")

The above produces the following: This is a margin note

You can place just about anything in this that you would normally put in markdown so go crazy.

Authors

In data/ add a file titled authors.toml and in this place entries as follows:

In this include the following info (replacing in the obvious places).

[Edward]
  name =  "Edward"
  bio = "This is my super awesome bio and stuff."
  location =  "Normal, Il"
  website =  "http:/www.example.com"
  thumbnail = "http://img11.hostingpics.net/pics/337670Tintin.jpg"
  twitter = "https://twitter.com/"
  github = "https://github.com/nstrayer/tuftesque/"
  
[Other Edward]
  name =  "Not the best edward"
  ...

Make sure that the key [Author Name] is exactly as the name appears in the author field in your post. No need for quotes in the key field in the .toml E.g. author = "Jane Doe" would correspond to [Jane Doe] in authors.toml.
